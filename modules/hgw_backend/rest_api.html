API Methods¶

POST /oauth2/token/¶

Obtain an OAuth2 authorization token to be used by the Source Endpoint to make request to the other API functions. The token will have a message:write scope, which is the one that allows to send messages, or a sources:read scope, which is the one that to read sources.

Query Parameters:
 
  • client_id – the oauth2 client id assigned to a Source Endpoint during the enrollment phase
  • client_secret – the oauth2 client secret assign during the enrollment phase
  • grant_type – type of grant used. In our case is always client_credentials

Success Response

HTTP/1.1 200 OK
Content-Type: application/json

{
    "access_token": "fNN8u8THUuG5KVkTRlTtxKe7mgF1aG",
    "token_type": "Bearer",
    "expires_in": 36000,
    "scope": "messages:write"
}
GET /v1/sources/¶

Gets the list of sources with their associated profiles

Request Headers:
 
  • Authorization – Bearer <oauth_2_access_token>
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 200 OK – The request was successfull
  • 401 Unauthorized – Unauthorized - The client has not provide a valid token or the token has expired
  • 403 Forbidden – Forbidden - The client token has not the right scope for the operation
  • 500 Internal Server Error – Internal Server Error - Something wrong happened

Success response

HTTP/1.1 202 OK
Vary: Authorization
Content-Type: application/json

[{
    "source_id": "WeiMaK8pjMQ6B9qxDRFm00EcFyi1NyFN",
    "name": "SOURCE_ENDPOINT",
    "profile": {
        "code": "PROF_001",
        "version": "v0",
        "payload": "[{\"clinical_domain\": \"Laboratory\"}]"
    }
}]
GET /v1/sources/{str: source_id}¶

Gets the source with id source_id and its associated profiles

Request Headers:
 
  • Authorization – Bearer <oauth_2_access_token>
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 200 OK – The request was successfull
  • 401 Unauthorized – Unauthorized - The client has not provide a valid token or the token has expired
  • 403 Forbidden – Forbidden - The client token has not the right scope for the operation
  • 404 Not Found – Not Found - The source with the specified id was not found
  • 500 Internal Server Error – Internal Server Error - Something wrong happened
Parameters:
  • source_id – the id of the source to get

Success response

HTTP/1.1 202 OK
Vary: Authorization
Content-Type: application/json

{
    "source_id": "WeiMaK8pjMQ6B9qxDRFm00EcFyi1NyFN",
    "name": "SOURCE_ENDPOINT",
    "profile": {
        "code": "PROF_001",
        "version": "v0",
        "payload": "[{\"clinical_domain\": \"Laboratory\"}]"
    }
}
POST /v1/messages/¶

Creates (i.e., sends) a new message. The Source Endpoint must include the channel_id the message refers to and the payload which is an encrypted string. To use this endpoint the client needs a token with messages:write scope

Form Parameters:
 
  • channel_id – The channel id the message refers to
  • message – The encrypted message
Request Headers:
 
  • Authorization – Bearer <oauth_2_access_token>
  • Content-Type – multipart/form-data
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 200 OK – Success - Message sent correctly
  • 400 Bad Request – Bad Request - Missing parameters or payload not encrypted
  • 401 Unauthorized – Unauthorized - The client has not provide a valid token or the token has expired
  • 403 Forbidden – Forbidden - The client token has not the right scope for the operation
  • 500 Internal Server Error – Internal Server Error - Something wrong happened sending the message (e.g., broker was unreachable)

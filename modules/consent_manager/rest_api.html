API Methods¶

POST /oauth2/token/¶

Obtain an OAuth2 authorization token to be used to make request to the other API functions

Parameters:
  • client_id – the oauth2 client id assigned during the enrollment phase
  • client_secret – the oauth2 client secret assign during the enrollment phase
  • grant_type – type of grant used. In our case is alway client_credentials

Success Response

HTTP/1.1 200 OK
Content-Type: application/json

{
    "access_token": "fNN8u8THUuG5KVkTRlTtxKe7mgF1aG",
    "token_type": "Bearer",
    "expires_in": 36000,
    "scope": "consent:read consent:write"
}
POST /v1/consents/¶

Creates a new Consent/Channel which is set in a PENDING status until the user confirms it

Parameters:
  • source_id – the id of the Source that will send data
  • destination_id – the id of the Destination that will receive the data
  • profile – the Profile object
  • person_id – the person id
Request Headers:
 
  • Authorization – Bearer <oauth2_authorization_token> The authorization token obtained before
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 201 Created – The Consent/Channel has been created correctly
  • 400 Bad Request – Something wrong with the parameters
  • 500 Internal Server Error – Something wrong happened

Success Response

HTTP/1.1 201 Created
Content-Type: application/json

{
    "consent_id": "2Eko7Zw39wPWVNaBbwClzbFpjJ97nHHb",
    "confirm_id": "xdv5jlQiWNW3ZaFMvmyVev5A0AGOZEHC"
}
GET /v1/consents/¶

Get a list of information about the Consents/Channels. In particular it returns the destination id and the status.

Request Headers:
 
  • Authorization – Bearer <oauth2_authorization_token> The authorization token obtained before
Response Headers:
 
  • Content-Type – application/json
Status Codes:
  • 200 OK – Everything ok
  • 500 Internal Server Error – Something wrong happened

Success Response

HTTP/1.1 200 OK
Content-Type: application/json

[{
    "status":"PE",
    "destination_id":"vnTuqCY3muHipTSan6Xdctj2Y0vUOVkj"
}]
